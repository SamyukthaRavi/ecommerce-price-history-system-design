# Trade-offs & Design Decisions

## Decision: Use Time-Series Database

### Alternative
Relational database

### Why Chosen
Optimized for time-based queries and efficient storage

---

## Decision: Subscriber-only Feature

### Alternative
Show to all users

### Why Chosen
- Reduces system load
- Adds value to subscription
- Controls feature rollout

---

## Decision: Show Only During Sale Events

### Alternative
Always available

### Why Chosen
- Reduces unnecessary usage
- Focuses on high-impact moments
